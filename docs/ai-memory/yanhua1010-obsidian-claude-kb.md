## Summary
@yanhua1010 分享的 Obsidian+Claude KB 工作流，结合 Karpathy "编译" 隐喻。三层架构：原料/摘要/沉淀，配合 CLAUDE.md+index.md+log.md 三个核心文件。知识库的本质是将原始内容"编译"为可检索的结构化资产。

## Key Concepts
- **Obsidian KB** — Obsidian 作为 KB 前端，Claude 作为处理引擎
- **Three Layer Architecture** — 原料→摘要→沉淀的递进处理管道
- **Compile Metaphor** — 借用 Karpathy 隐喻：LLM = 编译器，KB = 目标文件
- **CLAUDE.md** — 项目级 Claude 指引文件

## Architecture
```
原料层 (Raw)    → 原始资料收集
摘要层 (Summary) → LLM 提炼摘要
沉淀层 (Synth)  → 深度知识沉淀
```

核心文件：`CLAUDE.md` + `index.md` + `log.md`
