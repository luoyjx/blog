## Summary
Anthropic Claude Code产品负责人Cat Wu提出AI时代产品管理的四个关键转变。传统PM方法论建立在"底层技术大致稳定"的假设之上，但当前模型能力指数级进步已使这一前提失效。PM角色必须从"规划控制者"转向"实验催化剂"与"能力冲浪者"。核心工作流围绕Claude.ai、**Claude Code**和Cowork三个工具展开，实现从想法到原型数小时内完成。PM工作已从"前端定义确定性"转向"加速发现与紧**Feedback Loop**"。

## Key Concepts
- **Rapid Prototyping** - AI驱动的快速原型构建，从想法到可用原型仅需数小时
- **Sprint Planning** - 短Sprint替代长路线图
- **Model Capability Curve** - 模型能力持续上移，旧约束快速失效
- **Demo Driven Development** - 演示与评估优先于文档
- **Feedback Loop** - 紧反馈循环加速决策
- **Side Quest Experimentation** - 鼓励自发实验

## Detailed Content

### Cat Wu的工作流
Wu的工作围绕三个互补工具：**Claude.ai**作为战略思考伙伴；**Claude Code**构建原型、评估和脚本；Cowork管理收件箱、待办和日程。业界同行报告类似模式："从想法到可用原型只需数小时"。

### 四个关键转变

#### 1. 规划短Sprint，而非长路线图
高速小实验胜过固定规划。**Sprint Planning**鼓励**Side Quest Experimentation**式自发实验，速度成为核心竞争力。团队灵感可立即原型、内部dogfooding，若用户认可便快速上线。

#### 2. 演示与评估优先于文档
**Rapid Prototyping**将idea-to-prototype时间压缩至数小时，保真度远高于文字描述。"先构建再决策，而非先文档再开发"——**Demo Driven Development**成为新常态。

#### 3. 随新模型发布重新评估特性
每次模型更新都重测此前"太难"的功能清单，同时移除已过时的临时脚手架。**Model Capability Curve**持续上移，PM需养成"定期重置"习惯。

#### 4. 坚持做最简单的事
Agent系统里复杂性会指数级放大失败风险。优先"能work的最简方案"，让模型进步自然淘汰冗余。曾精心设计的system prompt在Opus 4.6上已精简20%。

### 组织影响
数据科学、财务、营销、法务和设计团队都独立开始使用**Claude Code**，整个组织以加速速度运转。
