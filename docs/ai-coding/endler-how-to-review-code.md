## Summary
Matthias Endler 20年代码审查经验总结。核心：审查不只是语法检查，是系统性思考（改动解决了什么问题）+ 人本协作（苏格拉底式提问、配对编程建信任）+ 标准把关（敢于说不，防技术债）。

## Key Concepts
- **Code Review** — 系统性审查，超越语法层面
- **Technical Debt** — "以后再修"陷阱的防范
- **Engineering Culture** — 代码审查作为集体学习机会

## Principles
| 原则 | 实践 |
|------|------|
| 系统思考 | 看改动解决了什么，影响了什么 |
| 命名 | 清晰命名减少认知负担，必须做对 |
| 坚守标准 | 敢于说不，给出理由 |
| 人本沟通 | 提问 > 否定；"我会这样做" |
| 多轮迭代 | 宏观 → 细节，不为速度牺牲质量 |
