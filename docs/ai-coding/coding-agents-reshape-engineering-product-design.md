## Summary

编程 Agent 从根本上改变了软件开发工作流，带来五大核心变化。传统 PRD（产品需求文档）作为工作流锚点已过时，但需求文档以新形式存在。开发瓶颈从实现转向审查，评估成为关键职能。跨越产品、工程和设计的通才受益最大，因消除了沟通开销。两种新兴角色浮现：Builder（用 Agent 创造）和 Reviewer（评估架构和产品合理性）。在执行成本降低的环境中，**Systems Thinking** 成为核心差异化竞争力。

## Key Concepts

- **Coding Agent** - 从根本上改变软件开发工作流的 AI 工具
- **PRD Evolution** - 传统产品需求文档形态的演变
- **Bottleneck Shift** - 开发瓶颈从实现转向审查
- **Generalist Advantage** - 跨领域通才在 Agent 时代的优势
- **Systems Thinking** - 执行成本降低时的核心差异化技能
- **Builder Reviewer Roles** - Agent 时代的两种新兴角色

## Detailed Content

### PRD 已死

传统产品需求文档作为工作流锚点已过时。**Coding Agent** 改变了从需求到实现的传统链路，但需求文档本身仍以新形式存在——形态和流程发生了根本变化。

### 瓶颈转移

**Bottleneck Shift**：开发瓶颈从**实现**转向**审查**。当 **Coding Agent** 大幅降低编码成本后，评估（evaluation）成为工程、产品和设计团队的关键职能。

### 通才优势

**Generalist Advantage**：跨越产品、工程和设计的通才受益最大。**Coding Agent** 消除了传统多角色沟通的开销，一个人可以完成过去需要多人协作的工作。

### 两种新兴角色

**Builder Reviewer Roles**：
- **Builder**：用 Agent 直接创造产品和功能
- **Reviewer**：评估架构决策和产品方向的合理性

### 系统思维的价值

**Systems Thinking** 在执行成本大幅降低的环境中成为核心竞争力。当"做出来"不再是瓶颈时，"做对的事情"变得更加重要。
