## Summary

@WSInsights 三年使用 Claude 迭代出的 18 个"肌肉记忆"级提示词，覆盖**投资人/知识工作者全工作流**：日规划、赛道研究、思路梳理、新概念学习、重大决策、写作、解死局、看项目、读研报、周复盘、内容创作、写开头、用户解剖、报价优化、技能缺口、砍废动作、个人 IP、销售文案。**核心思想**：提示词不是给 Claude 的指令，是**把自己模糊焦虑翻译成具体变量**的脚手架——逼自己在结构化框架内思考。**金句**：决策类（关键不是它替我决定，是它逼我把"害怕失败"翻译成"具体风险敞口"——前者是情绪，后者可以管理）；销售类（卖诊断不卖产品，成交率 +40%）；写作类（先 vomit 再 polish，速度三倍质量更高，因为第一稿敢写得更野）。**2026 新变量**：Claude Code 写完整项目 + MCP 接数据库/日历/邮件 → **提示词正在变成"智能体编排语言"**——你写的不是一段话，是一个工作流的指挥棒。**早点学会指挥，别等着被指挥。**

## Key Concepts

- **提示词肌肉记忆** — 不是理论，是每天睁眼就用的；从上千次对话里迭代出来的
- **把焦虑翻译成变量** — 提示词的真正价值是把模糊情绪结构化为可管理的变量
- **先 vomit 再 polish** — 闭眼写一坨垃圾不修改，AI 修改后再微调；第一稿敢写得更野
- **死亡笔记分析** — 看新项目先验尸：维生素/止痛药、市场时机、竞争盲区、收入幻觉、团队基因
- **卖诊断不卖产品** — 销售文案 7 步：钩子+伤口+诊断+方案+证据+风险逆转+行动；成交率 +40%
- **杠杆动作** — 解死局的关键问题："改动哪个最小变量，能引发最大连锁反应？"
- **症状 vs 病根** — 拆问题先问"表面问题和根因差几层"
- **一句话死刑/一句话投资** — 看项目的双向极端测试
- **拿到对话资格** — 学新东西的目标不是成为专家，是 40 分钟拿到对话资格 + 嗅出谁在吹牛
- **凌晨三点的焦虑** — 用户解剖六问中最锋利的一刀：他们睡不着时在担心什么
- **反承诺建立稀缺性** — 报价时加"什么情况我不接"，反向提升稀缺性
- **假装学会陷阱** — 学新技能时主动问 AI"这个领域最常见的假装学会陷阱是什么"
- **提示词即编排语言** — 2026 后提示词不是一段话，是一个工作流的指挥棒
- **72 小时测试** — 决策框架：如果 72 小时后必须执行，我现在该准备什么

### 五个最锋利的设计模式

#### 1. "把焦虑翻译成变量"（#5 决策模板）

```
情绪 → 变量
害怕失败 → 风险敞口（金额、时长、可对冲）
"看起来对" → 反直觉点（这个决定可能错在哪）
拍脑袋 → 72 小时测试
```

**作者原话**：关键不是它替我决定，**是它逼我把"害怕失败"翻译成"具体风险敞口"——前者是情绪，后者可以管理。**

#### 2. "卖诊断不卖产品"（#18 销售模板）

```
钩子（场景共鸣）
  ↓
伤口（量化损失）
  ↓
诊断（你之前为什么没用）  ← 核心权力
  ↓
方案（我的不同）
  ↓
证据（具体数字）
  ↓
风险逆转（你几乎没损失）
  ↓
行动（why now）
```

**关键转换**：
- 不写"年化回报 X%"
- 写"你过去三年的基金回报，跑输通胀了吗？"
- **从"我的能力"变为"你的诊断"**

#### 3. "死亡笔记分析"（#8 看项目）

```
[ 8 个问题 ]
1. 维生素 vs 止痛药
2. 为什么是现在（三年前太早 / 三年后太晚的证据）
3. 竞争盲区（创始人没提的替代方案）
4. 收入幻觉（第 1 个客户 vs 第 100 个客户）
5. 团队基因（背景 vs 核心能力的匹配度）
6. 一句话死刑（必拒的最硬理由）
7. 一句话投资（必投的最赌的点）
8. ↑ 让 BP 浓缩成一段话喂给 AI
```

**作者实战**：上周 AI Agent 项目，AI 指出"团队没有 infra 背景但产品核心是低延迟调度"——追问，创始人卡壳，**省了 200 万**。

#### 4. "凌晨三点的焦虑"（#13 用户解剖）

> 凌晨三点的焦虑：他们睡不着时在担心什么？

这是用户研究里**最锋利的一刀**——白天的伪装是表演，凌晨三点是真实。

**作者实战**："华尔街财经" IP 受众痛点不是"信息少"，**是"信息多但不会筛"**。这一刀决定了所有内容的角度。

#### 5. "假装学会陷阱"（#15 技能缺口）

> 这个领域最常见的"假装学会"陷阱是什么？

**反向 prompt**——主动问 AI 自己可能踩什么坑。这是 Dunning-Kruger 的预防接种。

### 元层洞察：提示词的本质

#### 提示词不是给 AI 的指令

> 它是把**自己模糊焦虑**翻译成**结构化变量**的**脚手架**。

```
旧模式：用户脑子乱 → AI 给乱答案
新模式：用户脑子乱 → 提示词强制结构化思考 → AI 在结构内输出
```

#### "肌肉记忆"的标准

不是用过一两次，而是：
- 每天用
- 不需要思考
- 改括号里的内容立刻生效
- 被反复迭代过（上千次对话沉淀）

#### 2026 的新变量

```
2024 提示词：一段文字
   ↓
2025 提示词 + Skills：可复用的工作模板
   ↓
2026 提示词 + Claude Code + MCP：智能体编排语言
   ↓
你写的不是话，是工作流的指挥棒
```

> **早点学会指挥，别等着被指挥。**

### 战略启示

#### 1. 这 18 个提示词的真正价值

不是 18 个具体模板，而是 18 个**思考结构**。换不同领域（律师/医生/咨询/创业者），结构可以平移：
- "死亡笔记 8 维"可以用于看任何项目/方案
- "决策压力测试"对所有人都通用
- "凌晨三点的焦虑"是用户研究的元工具

#### 3. 与"理解不可外包"呼应

Karpathy 说"理解无法外包，但思考过程可以外包"——这 18 个提示词正是**外包思考过程**的具体落地：
- 提示词提供思考框架
- AI 提供推理算力
- 用户保留最终判断和行动

> 用户保留的是"判断什么值得做"和"承担风险"——这两件事 Claude 永远不能替你做。
